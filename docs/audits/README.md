# Audits

## Objectif

Centraliser les audits Loop Engine.

Les audits documentent l'état du projet à un moment donné.

Ils ne remplacent pas les documents d'architecture actifs.

## Règle de lecture

En cas de contradiction :

1. les documents `docs/architecture/` font autorité ;
2. le changelog fait autorité sur l'historique livré ;
3. les audits les plus récents priment sur les audits plus anciens ;
4. les audits anciens restent utiles comme contexte historique.
