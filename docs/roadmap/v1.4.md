# Loop Engine V1.4

## Objectif

Préparer l'exploitation des sorties JSON validées.

V1.4 doit rester en lecture seule.

---

## Axes possibles

### Option A — Dashboard local read-only

Afficher :

- état workspace ;
- projets dirty ;
- validations configurées ;
- candidat roadmap sélectionné ;
- statistiques roadmap.

Avantage : exploite directement les contrats JSON existants.

Risque : peut introduire trop tôt une UI.

---

### Option B — n8n read-only

Créer un workflow qui consomme :

- `summary --json`
- `next <project> --json`

Avantage : utile pour alertes et tableau de bord.

Risque : nécessite un environnement n8n stable.

---

### Option C — OpenClaw read-only

Utiliser les JSON pour alimenter une couche assistant.

Avantage : rapproche Loop Engine de l'orchestration IA contrôlée.

Risque : peut pousser trop tôt vers l'automatisation.

---

### Option D — Roadmap Reader V1.4

Améliorer encore la lecture roadmap :

- sections explicites ;
- candidats par statut ;
- meilleur résumé ;
- priorités écrites.

Avantage : améliore le coeur du moteur.

Risque : parsing Markdown plus fragile.

---

## Recommandation

Commencer par **Option D — Roadmap Reader V1.4**.

Raison :

- aucun nouvel outil externe ;
- pas d'UI prématurée ;
- pas d'automatisation ;
- renforce les données déjà consommées par JSON ;
- prépare mieux dashboard, n8n et OpenClaw.

---

## Premier lot recommandé

Ajouter une synthèse roadmap dans `ProjectSnapshot` :

- nombre total de candidats actifs ;
- nombre de candidats terminés ;
- nombre de candidats sélectionnables ;
- présence ou non d'un candidat bloqué.

Cette synthèse doit être calculée dans `intelligence/`, pas dans les commandes.
