# Loop Engine V1.7

## Objectif

Préparer les intégrations read-only autour des contrats JSON stabilisés.

V1.7 ne doit pas introduire d'automatisation.

## Candidats d'intégration

### n8n

Usage cible :

- lire `summary --json` ;
- détecter les projets dirty ;
- afficher les candidats sélectionnés ;
- notifier sans modifier.

### OpenClaw

Usage cible :

- lire `context --json` ;
- lire `next --json` ;
- préparer une session humaine ;
- ne jamais lancer d'action autonome.

### Dashboard local

Usage cible :

- afficher workspace ;
- afficher roadmap summary ;
- afficher priority/status/kind ;
- rester read-only.

## Recommandation

Commencer par n8n read-only.

Raison :

- pas besoin d'UI dédiée ;
- exploite déjà les JSON compacts ;
- utile pour supervision locale ;
- garde l'humain en contrôle.

## Premier micro-lot recommandé

Créer une documentation d'intégration n8n read-only :

- commandes à appeler ;
- fréquence raisonnable ;
- données utiles ;
- garde-fous ;
- limites V1.7.
